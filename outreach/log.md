# Outreach Response Log

Founder updates this file as outreach moves. The format is a single
table; replace placeholder rows with real data as sends happen.

Outcome key:
- `drafted` — Cowork has a draft ready, not yet personalised
- `sent` — founder pressed send
- `opened` — email-tracking pixel confirmed open (optional signal)
- `replied-pos` — substantive reply, conversation moving
- `replied-neg` — explicit "no thanks" / "not now"
- `silent` — 7+ days since send, no response
- `bounced` — bad email address / bounce
- `meeting` — call scheduled or held
- `pilot` — pilot agreement (verbal or written); upgrade to
  `pilot-written` once it's in writing

## What the application's "design partner" answer claims today

Based on the live counters above. Two phrasings, pick the higher
that's literally true:

- **Lower bound (always available):** "outreach in flight to 20
  design-partner targets across AI-coding, AI-infra, and
  verification-focused teams."
- **Upper bound (only when counters justify):** "active pilots
  with `[N]` teams" — only when `Pilots in writing ≥ N`.

Do not gild. The application loses more credibility from a single
overclaimed pilot than it gains from gilding.
