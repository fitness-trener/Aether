## 1. AI-generated code volume — where the market lives

### What's measurable

The proxies for "how much production code is now AI-generated"
that have public-source-citable numbers as of mid-2025:

- **Copilot acceptance rate.** GitHub has published in
  Octoverse-class reports (and in their developer-blog posts)
  that ~30%–50% of code suggestions in a typical Copilot session
  are accepted. The number varies by language and developer
  experience. Source: GitHub's official Copilot research posts
  (cite the most recent at submission time).
- **Cursor / IDE-agent usage growth.** Anysphere has been public
  about Cursor's user base hitting hundreds of thousands of
  active engineers in 2024–2025. The exact number at submission
  time is `[TBD — pull from their most recent funding-round
  announcement]`.
- **Replit Agent / coding-agent traffic.** Replit publishes
  monthly active developer counts in their investor decks.
  Number at submission time: `[TBD — Replit's most recent
  shareholder letter]`.
- **Sourcegraph / Cody enterprise adoption.** Sourcegraph
  publishes their enterprise customer count and the share that
  uses Cody. Cite at submission time. `[TBD]`

### What's NOT measurable

The number we'd actually want — *what fraction of code currently
running in production was generated by an AI agent* — has no
authoritative public source. Every cited figure ("X% of new code
is AI-generated") traces back to either an internal-platform
estimate (which can't be independently verified) or a survey of
self-reporting developers (which has well-known measurement
bias).

The application **does not claim** a specific "X% of production
code is AI-generated" figure unless we can hyperlink a public
study. The Path-B framing in `application_v8.md` is anchored on
trajectory ("dominant mode within 24–36 months") rather than a
2026 spot estimate — defensible because the trajectory is
consensus-level, the spot estimate is contested.

## 2. The structural bet

### Bottom-up — AI-coding companies as the wedge

The wedge market is the ~100 companies whose product is the
AI-coding agent itself. The 20-company target list at
`outreach/targets.md` covers approximately the top 20 by
revenue + funding + technical credibility.

A defensible (not "made up") bottoms-up:

- ~100 companies in the AI-coding category globally as of mid-2025.
  Cite source: PitchBook's "AI Software" segment classification
  + Crunchbase's AI-coding tags. `[TBD — pull the actual count
  from the most recent PitchBook export]`.
- Each company averages roughly 50 internal engineers + a growing
  pool of customer-deployed agent seats. The internal-engineer
  number is bound by Series-A-to-D headcount norms (well-
  documented in `[TBD — cite a recent venture-tools-funding
  report]`).
- Aether's pricing target for Phase 2 (Years 2–3): the per-agent-
  seat or per-execution model comparable to JetBrains' Qodana
  inspection toolchain. JetBrains-public pricing for similar
  product surfaces: $X / engineer / month. `[TBD — JetBrains
  public price sheet at submission time]`.

The honest top-line: even if Aether captures only 10% of this
wedge at $50/seat/month over 50 seats, the unit economics get to
seven figures of ARR per design partner within 24 months. The
real upside is the Phase 3 (enterprise compliance) layer — but
that layer is too forward-looking to size honestly today.

### Top-down — developer-tools attach

The market Aether attaches to is "developer tools that ship into
the SDLC of AI-assisted code." The publicly-cited adjacent
markets:

- **Static analysis / SAST market** (SonarQube, Snyk Code,
  Semgrep, Coverity-class) — `[TBD — Gartner's most recent
  Critical Capabilities for AST in $B]`.
- **Programming language tooling market** (JetBrains, GitHub
  Copilot, Tabnine) — `[TBD — Gartner Magic Quadrant for ALM

The structural argument matters more than the spot TAM: the
language that defines architectural correctness for AI-generated
production code captures a permanent position analogous to what
TypeScript captured for JavaScript. TypeScript's outcome is
public: Microsoft does not directly monetise the language; the
tooling-ecosystem capture around it is what compounds. Aether's
five-year arc bets on a comparable shape.

## 3. Why we do not include a TAM number in the application

YC partners discount inflated TAMs. The numbers above are real but
small — the wedge market is ~100 companies, not 100,000. The honest
phrasing in `application_v8.md` is "bottoms-up estimate over the
24-month conversion window," not "$N billion TAM."

A partner asking "but how big can this get?" gets the structural
argument: programming languages aren't a TAM-bound business
once they reach critical mass — JavaScript, Python, Rust, Java
each captured a structural position that produced ecosystem
markets larger than any single-product TAM. Aether's bet is the
same shape, calibrated to a smaller-but-still-meaningful wedge
(AI-generated production code).

## 4. Pricing model (informed by adjacent comparables)

### Phase 1 (Years 1–2) — Free / OSS

The aether-lang core is MIT-licensed. Free to install, free to
use, free to fork. The benchmark + the SDK + the LSP all ship
with the open-source release. This is the move that turns
distribution into the moat.

Revenue: zero. Funded by YC + seed.

### Phase 2 (Years 2–3) — Hosted toolchain SaaS

The commercial product is hosted Aether for teams: shared
playground, persistent fix-loop transcripts, the live-LLM
fix-loop wired into Anthropic / OpenAI / Bedrock at metered
billing, plus collaboration features.

Pricing comparables (cite at submission time):

- **JetBrains Qodana** (inspection toolchain for human engineers):
  $X / engineer / month. `[TBD — JetBrains' published 2025
  price]`. Our analog: per-agent-seat at a multiple of the
  Qodana number, because each agent generates dramatically more
  code than each human.
- **GitHub Advanced Security** (which prices on a per-committer
  basis for SAST + secret scanning): `[TBD — GitHub's most
  recent enterprise SKU price]`.

Honest target ACV per design partner in this phase: $50k–$200k.
Number of design partners by end of Year 2: 10–20. Total Phase 2
ARR ceiling: low seven figures.

### Phase 3 (Years 3+) — Enterprise architectural-integrity attestation

The high-margin product. CISOs at AI-using enterprises buy
compile-time evidence that AI-generated code in production
satisfies their architectural / regulatory promises.

Pricing comparables:

- **HashiCorp Vault Enterprise** (the "policy compliance for
  infrastructure" analog): `[TBD — pull from HashiCorp's most
  recent 10-K]`.
- **Snowflake Governance Edition** (compliance-tier pricing in
  the data-platform space): `[TBD — Snowflake recent enterprise
  ACV]`.

Honest target ACV in this phase: $200k–$1M.

The math that gets to material outcomes: 50 enterprise customers
× $500k ACV = $25M ARR. Five years to that number is realistic;
two is not. The application does not claim two.
